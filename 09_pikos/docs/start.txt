
    ПРОЦЕСС ЗАПУСКА 32 БИТНОЙ PIKOS
------------------------------------------------------------------------

    Boot-этап

        + Переключение в VESA 2.0 максимально доступный видеорежим 16 bit
        + Запись информации о видеорежиме в Conventional Memory
        + Инициализация ProtectedMode, TSS: SwitchProtectedMode
        + Переключение задачи LTR, заполнение TSS
        + Инициализация PIC        
        + Определение размера памяти        
        + Резервирование области для описателей страниц
        + Установка страничной адресации на текущий размер памяти
        + Установка обработчиков прерываний        
        + Инициализация таймера

    Этап обнаружения жестких дисков
    
        - Обнаружение наличия жестких дисков в системе IDE-mode
        - Системный драйвер PIO жестких дисков ATA
        - Работа с FAT16/32, загрузка файлов в память
        
        - Возможно. Инициализация PS/2 мыши на будущее

    After-этап

        - Загрузка программ в память

------------------------------------------------------------------------

    НОТАЦИИ

    - Все имена переменных пишутся так variable_name
    - Все имена функции пишутся FunctionName
    - У каждой функции подписывать область их действия, к примеру

        call    mm.Malloc

    - Каждая функция находится в отдельной файле

------------------------------------------------------------------------

    СТРУКТУРА

        pikos.asm                       Основной файл
        fs/                             Файловые системы, доступ к дискам
        mm/                             Модули управлению памятью
        app/                            Приложения
        vesa/                           VESA-функции
        

------------------------------------------------------------------------

    ЧТО ДОЛЖНА УМЕТЬ СИСТЕМА
    
        - Работать с графикой - рисование примитиров
        - Работать на низком уровне с дисковой подсистемой
        - Читать и писать в FAT16/32
        - Обрабатывать прерывания Timer, Keyboard и другие
        - Инициализировать PCI
        - Работать с Ethernet
        - Читать и писать в системный реестр
        - Уметь запускать приложения
        - Приложения COM должны быть максимальным образом совместимы 
          с DOS. Кроме сегментации. Но должны работать Int 10h, 13h, 21h

    Система должна быть "прослойкой" между запускаемыми приложениями
    и "железом". Это самое главное.
    
    Пока что приложения будут только COM с tiny-моделью памяти.

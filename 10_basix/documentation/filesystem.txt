Структура файловой системы

    /initrd/*               данные с диска initrd

Как устроены вызовы

    * при открытии файла, смотрится на его путь, в зависимости
      от зарегистрированных путей в системе, выбирается драйвер
      файловой системы

    * передается необходимой процедуре

Файловые дескрипторы

    * в системе не может быть открыто более 256 файлов
    * каждый файловый дескриптор ссылается на собственную
      структуру

      file_description[ N ] = M
      internal_fat12[ M ] = <struct>

      но в итоге каждый файл имеет одну и ту же структуру
      FILE (vfs)

    * любой процесс, не относящийся к kernel, пишет информацию
      в FILE (process_id). При закрытии процесса файл
      закрывается тоже
